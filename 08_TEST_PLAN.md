# テスト・検算仕様

## 1. 単体テスト

### 1.1 計算式

以下をテストする。

- 使用料単価
- 汚水処理原価
- 経費回収率
- 100％達成必要改定率
- 80％達成必要改定率
- 150円/m³達成必要改定率
- 有収率
- 水洗化率

### 1.2 例

```text
下水道使用料収入 = 80,000 千円
汚水処理費 = 100,000 千円
年間有収水量 = 500,000 m³

使用料単価 = 160 円/m³
汚水処理原価 = 200 円/m³
経費回収率 = 80%
100％達成必要改定率 = 25%
80％達成必要改定率 = 0%
150円/m³達成必要改定率 = 達成済み
```

## 2. ETLテスト

### 2.1 Discover

- 対象年度を指定するとsource_files候補が取得できる。
- 法適用の表19,20,32,40が取得候補に含まれる。
- 法非適用の表10,26,32,40が取得候補に含まれる。

### 2.2 Download

- ファイルが `data/raw` に保存される。
- SHA256が保存される。
- 2回目実行では同一ファイルを再ダウンロードしない。

### 2.3 Import

- 生セルが `raw_stat_cells` に保存される。
- 団体名、事業種別、表番号、行番号、列番号が保存される。
- 数値変換できないセルは `value_numeric` がNULLになる。

### 2.4 Normalize

- 標準項目が `annual_financials` に保存される。
- 欠損値がある場合でも処理全体は落ちない。
- 欠損フラグが付く。

## 3. 検算テスト

### 3.1 経費回収率

国交省資料や自治体の経営比較分析表等に掲載されている経費回収率と、サイト計算値が概ね一致することを確認する。

許容誤差：

- 通常: ±0.2ポイント以内
- 単位丸めがある場合: ±0.5ポイント以内
- 法適用移行年度・打切決算: warning扱い

### 3.2 使用料単価・汚水処理原価

同様に自治体資料・年鑑値と突合する。

## 4. UIテスト

- トップページが表示される。
- 自治体検索ができる。
- 自治体詳細ページが表示される。
- 計算値がNULLの場合、画面が落ちず「算定不可」と表示される。
- 免責文言が表示される。
- データ根拠が表示される。

## 5. 異常値テスト

以下をテストデータで確認する。

- 有収水量0
- 汚水処理費0
- 下水道使用料収入0
- 経費回収率300％超
- 前年度比50％超の増減
- 法適用移行年度フラグ
